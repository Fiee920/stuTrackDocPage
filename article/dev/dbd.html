1.user ​

1.1基本信息 ​

该表为系统用户表，用于存储系统用户的基本信息。

1.2表设计 ​

字段名字段类型字段说明非空备注
uidINT唯一主键|用户标识T自增主键
usernameVARCHAR(32)用户名T5-16位字母/数字
passwordVARCHAR(128)密码Tmd5加密
nameVARCHAR(32)姓名
emailVARCHAR(64)邮箱见补充说明
role_idINT用户权限idT见补充说明
create_timeDATETIME创建时间T
update_timeDATETIME更新时间T
picVARCHAR(128)用户头像
is_deletedINT帐号状态T0-正常 1-注销

1.3补充说明 ​

  1. 该表与stuInfo表不冲突，仅为系统用户信息，主要用于权限校验。
  2. email字段赋值说明：默认值：default@default.com
  3. role_id字段赋值说明：8-超级管理员 4-教师账号 11-综合部账号 12-团委账号 13-公管会账号 14-学习部账号 15-通用账号
  4. stutrack V1 Alpha仅包含role_id值为8的用户，用户头像不存在。
  5. 账号权限说明：
    • 超级管理员拥有所有权限 （用户管理权限暂未开发）
    • 教师账号、综合部账号限制用户管理权限
    • 团委、公管会、学习部及通用账号完全限制文件流转、德育分中心，限制部分学生表字段（分部门）

1.4快速配置 ​

sql
-- 创建表
USE stutrack;
CREATE TABLE IF NOT EXISTS user (
    uid INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(32) NOT NULL,
    password VARCHAR(128) NOT NULL,
    name VARCHAR(32),
    email VARCHAR(64) DEFAULT 'default@default.com',
    role_id INT NOT NULL,
    create_time DATETIME NOT NULL,
    update_time DATETIME NOT NULL,
    pic VARCHAR(128),
    is_deleted INT DEFAULT 0
);
-- 插入用户数据（默认账号密码均为admin，建议及时修改密码）
USE stutrack;
INSERT INTO user (username, password, role_id, create_time, update_time, is_deleted)
VALUES ('admin', '21232f297a57a5a743894a0e4a801fc3', 8, NOW(), NOW(), 0);

2.stuInfo ​

2.1基本信息 ​

该表为学生信息表，包含了学生学号、姓名、班级、年级、性别等多项数据。

2.2表设计 ​

字段名字段类型字段说明非空备注
stu_uidINT学生表自增主键T
stu_idCHAR(9)学号T
stu_nameVARCHAR(32)姓名T
group_typeCHAR(4)组T本科,硕士
gradeCHAR(4)年级T
subjectVARCHAR(30)专业
clazzVARCHAR(16)班级T
genderCHAR(2)性别T男,女
phoneVARCHAR(128)电话号码分配更多字节应对错误数据
id_cardVARCHAR(128)身份证号码分配更多字节应对错误数据
ethnicityVARCHAR(32)民族
poliVARCHAR(16)政治面貌见补充说明
apartmentVARCHAR(16)公寓号0-未分配
roomVARCHAR(4)寝室号0-未分配
attendance_statusBOOLEAN走读状态1-走读 0-住宿
enrollment_statusVARCHAR(16)在籍状态见补充说明
birthplaceVARCHAR(64)籍贯
homeVARCHAR(64)家庭所在地
home_fullVARCHAR(128)详细居住地址
sos_phoneVARCHAR(128)紧急联系方式分配更多字节应对错误数据
sos_relVARCHAR(16)紧急联系人关系
scoreFLOAT德育分分数T默认值为80.0
仅保留一位小数
score2FLOAT第二课堂积分T默认值为0
仅保留一位小数
is_specialBOOLEAN是否为特殊学生
create_timeDATETIME创建时间T
update_timeDATETIME修改时间T
noteVARCHAR(512)备注

2.3补充说明 ​

  1. stu_uid字段为学生信息自增主键，与user表中的uid不相同。
  2. poli字段赋值说明：群众, 共青团员, 中共预备党员, 中共党员, 未知
  3. enrollment_status字段赋值说明：正常在校, 参军, 休学, 延毕, 其他

2.4快速配置 ​

sql
-- 创建表
USE stutrack;
CREATE TABLE IF NOT EXISTS stuInfo (
    stu_uid INT PRIMARY KEY AUTO_INCREMENT,
    stu_id CHAR(9) NOT NULL,
    stu_name VARCHAR(32) NOT NULL,
    group_type CHAR(4) NOT NULL,
    grade CHAR(4) NOT NULL,
    subject VARCHAR(30) NOT NULL,
    clazz CHAR(5) NOT NULL,
    gender CHAR(2) NOT NULL,
    phone CHAR(11),
    id_card CHAR(18),
    ethnicity VARCHAR(32),
    poli VARCHAR(16),
    apartment VARCHAR(2),
    room VARCHAR(4),
    attendance_status BOOLEAN,
    enrollment_status VARCHAR(16),
    birthplace VARCHAR(64),
    home VARCHAR(64),
    home_full VARCHAR(128),
    sos_phone CHAR(11),
    sos_rel VARCHAR(16),
    score FLOAT DEFAULT 80.0,
    score2 FLOAT DEFAULT 0,
    is_special BOOLEAN,
    create_time DATETIME NOT NULL,
    update_time DATETIME NOT NULL,
    note VARCHAR(512)
);

3.oprRec ​

3.1基本信息 ​

该表为德育分分数操作记录表，记录了每一次对分数操作的各种详细信息。

3.2表设计 ​

字段名字段类型字段说明非空备注
opr_idINT操作记录idT自增主键
opr_timeDATETIME操作时间T
opr_scoreFLOAT操作分数T见补充说明
opr_userINT操作人T与user表uid对应
opr_reasonVARCHAR(512)操作原因T
opr_target_stuCHAR(9)被操作用户T见补充说明
target_stu_scoreFLOAT被操作用户最新分数
statusINT该记录是否生效T见补充说明
stu_nameVARCHAR(32)学生姓名

3.3补充说明 ​

  1. opr_score字段赋值说明：需提交正数/负数，且必须为0.5的整数倍。数据库中不做校验，需在Java代码中完成校验。
  2. opr_target_stu字段赋值说明：与stuInfo表中的stu_id属性对应即可。
  3. status字段赋值说明：0-撤销, 1-通过, 2-锁定，8-审核中, 9-拒绝（当前版本不激活）

3.4快速配置 ​

sql
-- 创建表
USE stutrack;
CREATE TABLE oprRec (
    opr_id INT PRIMARY KEY AUTO_INCREMENT,
    opr_time DATETIME NOT NULL,
    opr_score FLOAT NOT NULL,
    opr_user INT NOT NULL,
    opr_reason VARCHAR(512) NOT NULL,
    opr_target_stu CHAR(9) NOT NULL,
    target_stu_score FLOAT,
    status INT NOT NULL,
    stu_name VARCHAR(32)
);

4.file_upload ​

4.1基本信息 ​

该表为文件上传表，用于储存文件名、存放位置等各项信息。

4.2表设计 ​

4.3补充说明 ​

  1. type字段赋值说明：1-名册, 2-德育分, 3-文件流转, 4-第二课堂积分
  2. file_md5字段用于存储最终文件的MD5值，当前版本暂未开发，无用。
  3. file_base64字段用于存储文件的BASE64编码，便于使用BASE64解码器还原源文件内容，暂未开发。(由于VARCHAR存放量较小，已废除)
  4. file_data字段使用BLOB（Binary Large Object）数据类型来保存文件的原始信息。暂未开发。

4.4快速配置 ​

sql
-- 创建表
USE stutrack;
CREATE TABLE file_upload (
  file_id INT AUTO_INCREMENT PRIMARY KEY,
  target_user INT NOT NULL,
  create_time DATETIME NOT NULL,
  file_name VARCHAR(512) NOT NULL,
  file_path VARCHAR(512) NOT NULL,
  type INT NOT NULL,
  file_md5 VARCHAR(128),
  file_data BLOB,
  note VARCHAR(512)
);

6.discipline ​

6.1基本信息 ​

该表为通报预警表，用于存储学生违纪和通报预警信息。

6.2表设计 ​

字段名字段类型字段说明非空备注
idINT申报记录idT自增主键
stu_idCHAR(9)涉及学生学号T
stu_nameVARCHAR(32)涉及学生姓名T
typeVARCHAR(64)通报类型T
descriptionVARCHAR(512)通报描述
timeDATETIME操作时间T
user_idINT操作用户T与user表uid对应
activeCHAR(1)生效状态T1-有效 0-撤销

6.3补充说明 ​

无

6.4快读配置 ​

sql
-- 创建表
USE stutrack;
CREATE TABLE discipline (
    id INT PRIMARY KEY AUTO_INCREMENT,
    stu_id CHAR(9) NOT NULL,
    stu_name VARCHAR(32) NOT NULL,
    type VARCHAR(64) NOT NULL,
    description VARCHAR(512),
    time DATETIME NOT NULL,
    user_id INT NOT NULL,
    active CHAR(1) NOT NULL
