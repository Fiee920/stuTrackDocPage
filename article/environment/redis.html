Redis环境配置 ​

基于CentOS7做命令演示，根据实际使用的Linux发行版灵活调整。

1.下载Redis ​

官网：https://redis.io/

历史版本：http://download.redis.io/releases

2.使用SFTP工具上传到Linux服务器 ​

3.安装gcc ​

shell
yum install gcc-c++

因为Redis是基于C语言编写的，所以需要安装gcc

安装好后通过指令gcc -v查看gcc版本

4.解压安装包 ​

这里可以将程序移动到opt目录下（Linux默认的程序存放路径）

shell
mv redis-7.2.4.tar.gz  /opt

对Redis文件进行解压

shell
tar zxvf redis-7.2.4.tar.gz

5.进入Redis目录下，进行编译 ​

首先执行make命令，然后执行make install等待并确定所有都安装完成

Redis的默认安装路径：/usr/local/bin

6.将redis配置文件复制到bin目录下 ​

shell
mkdir redisconfig
cp /opt/redis-7.2.4/redis.conf redisconfig

之后再启动时，就使用此文件启动。

7.调整Redis设置 ​

redis.conf

shell
# 允许访问的地址，默认是127.0.0.1，会导致只能在本地访问。修改为0.0.0.0则可以在任意IP访问，生产环境不要设置为0.0.0.0
bind 0.0.0.0
# 守护进程，修改为yes后即可后台运行
daemonize yes 
# 密码，设置后访问Redis必须输入密码
requirepass 123321
# 监听的端口
port 6379
# 工作目录，默认是当前目录，也就是运行redis-server时的命令，日志、持久化等文件会保存在这个目录
dir .
# 数据库数量，设置为1，代表只使用1个库，默认有16个库，编号0~15
databases 1
# 设置redis能够使用的最大内存
maxmemory 512mb
# 日志文件，默认为空，不记录日志，可以指定日志文件名
logfile "redis.log"

8.启动redis-server服务 ​

通过制定配置文件启动服务：

shell
redis-server redisconfig/redis.conf

使用redis-cli连接测试

shell
redis-cli -p 6379

如果设置了登录密码，则需要进行认证：

shell
auth 密码

9.查看Redis进程是否开启 ​

需要新建一个会话，执行以下命令：

shell
ps -ef | grep redis

10.关闭redis服务 ​

shell
shutdown

11.设置Redis开机自启动 ​

首先新建一个系统服务文件：

shell
vi /etc/systemd/system/redis.service
shell
[Unit]
Description=redis-server
After=network.target

[Service]
Type=forking
ExecStart=/usr/local/bin/redis-server /usr/local/bin/redisconfig/redis.conf
PrivateTmp=true

[Install]
WantedBy=multi-user.target

然后重载系统服务

shell
systemctl daemon-reload

执行以下命令，可以让Redis开机自启动：

shell
systemctl enable redis

其他命令：

shell
# 启动
systemctl start redis
# 停止
systemctl stop redis
# 重启
systemctl restart redis
# 查看状态
systemctl status redis
